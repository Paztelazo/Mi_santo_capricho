# Parte 3 - Identificación, análisis y evaluación de 4 riesgos

## Riesgo 1 — Acceso no Autorizado al Sistema de Login
**Descripción:** Un atacante puede intentar vulnerar el sistema de autenticación mediante fuerza bruta o inyección SQL para acceder a las cuentas de administrador.  

**Causa Raíz:** Falta de límites de intentos y ausencia de protección CAPTCHA o multifactor.  

**Activo Afectado:** Base de datos de usuarios y credenciales.  

**Probabilidad:** Alta  

**Impacto:** Alto  

**Nivel de Riesgo:** **Crítico (Rojo)**  

**Controles Relacionados:**  
- **A.5.15 Control de Acceso:** Define la restricción a sistemas solo para usuarios autorizados.  
- **A.8.5 Autenticación Segura:** Implementar MFA y bloqueo ante intentos fallidos.  
- **A.8.25 Desarrollo Seguro:** Validar inputs para evitar inyecciones SQL.  

**Medidas Actuales:** Se utilizan contraseñas cifradas y un sistema básico de bloqueo tras varios intentos.  

**Medidas Recomendadas:** Implementar autenticación multifactor y auditorías de intentos de inicio de sesión.

---

## Riesgo 2 — Pérdida o Corrupción de la Base de Datos
**Descripción:** Fallas del servidor o errores humanos podrían causar pérdida de información crítica (usuarios, productos, pedidos).  

**Causa Raíz:** Copias de seguridad insuficientes o falta de verificación de restauraciones.  

**Activo Afectado:** Base de datos de pedidos, clientes y productos.  

**Probabilidad:** Media  

**Impacto:** Alto  

**Nivel de Riesgo:** **Alto (Naranja)**  

**Controles Relacionados:**  
- **A.8.13 Respaldo de Información:** Se deben generar backups automáticos y verificables.  
- **A.8.14 Redundancia:** Replicación de datos en servidores externos.  
- **A.7.14 Eliminación Segura:** Garantizar que los medios antiguos no contengan datos residuales.  

**Medidas Actuales:** Backup diario en un servidor local.  

**Medidas Recomendadas:** Replicación automática en la nube y validación mensual de restauraciones.

---

## Riesgo 3 — Fuga de Información de Clientes
**Descripción:** Un empleado con permisos amplios podría exportar información personal o vender bases de datos.  

**Causa Raíz:** Falta de control en los privilegios y ausencia de monitoreo sobre accesos internos.  

**Activo Afectado:** Datos personales (nombres, correos, direcciones, teléfonos).  

**Probabilidad:** Media  

**Impacto:** Alto  

**Nivel de Riesgo:** **Alto (Naranja)**  

**Controles Relacionados:**  
- **A.5.18 Derechos de Acceso:** Asignar privilegios mínimos necesarios.  
- **A.8.12 Prevención de Filtración de Datos:** Detectar descargas masivas o exportaciones no autorizadas.  
- **A.8.15 Registración:** Monitorear accesos y acciones sospechosas.  
- **A.6.8 Reporte de Incidentes:** Canal de notificación interna de fugas o anomalías.  

**Medidas Actuales:** Roles de usuario definidos, pero sin alertas activas.

**Medidas Recomendadas:** Implementar DLP (Data Loss Prevention) y auditoría de descargas internas.

---

## Riesgo 4 — Malware o Ransomware en el Sistema
**Descripción:** Un malware podría cifrar los archivos del servidor o inutilizar el sistema web, afectando la disponibilidad.  

**Causa Raíz:** Falta de controles de protección perimetral, actualizaciones atrasadas o navegación insegura por parte del personal.

**Activo Afectado:** Servidor web, base de datos y equipos del personal administrativo.

**Probabilidad:** Media  

**Impacto:** Alto  

**Nivel de Riesgo:** **Alto (Naranja)**  

**Controles Relacionados:**  
- **A.8.7 Protección contra código malicioso:** Uso de antivirus y herramientas de detección proactiva.  
- **A.8.8 Gestión de vulnerabilidades técnicas:** Escaneos periódicos y actualización de software.  
- **A.8.9 Gestión de configuración:** Mantener configuraciones seguras y actualizadas.  
- **A.8.20 Seguridad en redes:** Monitoreo del tráfico y filtrado de accesos no autorizados.  

**Medidas Actuales:** Antivirus local y firewall básico.  

**Medidas Recomendadas:** Implementar monitoreo centralizado, segmentación de red y copias offline ante ataques de ransomware.
